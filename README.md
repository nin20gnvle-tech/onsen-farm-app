# 農園業務Webアプリ（farm-app）

フルーツ農園の現場で起きている情報共有・記録の課題を、Webアプリで解決するためのプロジェクトです。  
「誰が・いつ・どこまで作業したか分からない」「在庫が手書き」「温湿度が手書き」「連絡が口頭/LINE依存」を、記録と共有の仕組みで改善します。

---

## 解決したい課題

- 誰が何を何時にどこまでやったのか分からない
- 在庫管理が手書き
- 温度湿度管理が手書き
- 現場の情報共有が分散（口頭・個別チャットなど）

---

## 主な機能（予定）

### 1. 作業ログ（タスク管理） ※優先
- 記録方式：開始/終了（タイマー）＋後入力（まとめ入力）両対応
- 権限：閲覧は全員、編集/削除は本人のみ（管理者は全件可）
- フィルタ：日付 / 圃場 / 作業者 / 作業種別 / ステータス

### 2. 在庫管理（入出庫）
- 在庫マスタ
- 入庫/使用/廃棄/棚卸の履歴
- 現在庫は履歴から自動集計

### 3. 温湿度ログ
- 手入力から開始（将来的にセンサー連携も想定）
- しきい値アラート（将来）

### 4. 全体チャット（画像/動画添付）
- 全体1部屋（general）
- 画像/動画添付（動画は最大60秒）
- 最初は数秒ポーリングで新着反映、将来的にWebSocketでリアルタイム化

### 5. 認証/権限（招待制）
- 管理者が招待URL（トークン）を発行
- 作業者は招待URLから初回登録
- 役割：管理者 / 作業者

---

## 技術スタック

- Frontend: React（予定）
- Backend: Laravel（API）
- DB: （開発）SQLite 〜 （本番）MySQL/PostgreSQL 想定
- Auth: Laravel Sanctum（予定）
- Realtime: Laravel Broadcasting + Reverb（予定）
- Storage: ローカル → S3互換ストレージ（将来）

## セットアップ（Backend / Laravel）

- Git

```bash
cd backend
composer install

2 環境変数の設定
cp .env.example .env

3. 起動
